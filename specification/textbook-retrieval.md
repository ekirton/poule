# Textbook Retrieval

Educational content retrieval system: HTML chunking, embedding computation, vector storage, similarity search, and MCP tool integration.

**Architecture**: [textbook-retrieval.md](../doc/architecture/textbook-retrieval.md)

---

## 1. Purpose

Define the textbook retrieval system: HTML chunker, education encoder, education storage, education RAG facade, MCP tool handler, and build pipeline for populating the education database from Software Foundations HTML.

## 2. Scope

**In scope**: `HTMLChunker` (HTML parsing, section-based chunking, metadata extraction), `EducationEncoder` (all-MiniLM ONNX wrapper, tokenization, encoding), `EducationStorage` (SQLite read/write for chunks and embeddings), `EducationRAG` (facade combining encoder, index, and storage for query-time retrieval), `education_context` MCP tool handler, build pipeline, data models.

**Out of scope**: Neural retrieval for Coq declarations (owned by neural-retrieval spec), MCP protocol handling (owned by mcp-server spec), slash command prompt authoring (owned by commands/), Docker deployment (owned by Dockerfile).

## 3. Definitions

| Term | Definition |
|------|-----------|
| Chunk | A segment of educational content from a Software Foundations HTML page, with prose text, extracted code blocks, and source attribution metadata |
| EducationEncoder | ONNX Runtime wrapper for the all-MiniLM-L6-v2 INT8 model that maps natural-language text to 384-dim L2-normalized embedding vectors |
| EducationRAG | Facade that combines the encoder, in-memory embedding index, and storage to answer educational queries |
| Volume | One of the seven Software Foundations books, identified by abbreviation: lf, plf, vfa, qc, secf, slf, vc |

## 4. Data Models

### 4.1 ChunkMetadata

| Field | Type | Constraints |
|-------|------|-------------|
| `volume` | string | Required; one of: `lf`, `plf`, `vfa`, `qc`, `secf`, `slf`, `vc` |
| `volume_title` | string | Required; human-readable volume name (e.g., "Logical Foundations") |
| `chapter` | string | Required; chapter name extracted from `<h1 class="libtitle">` |
| `chapter_file` | string | Required; HTML filename (e.g., `Basics.html`) |
| `section_title` | string | Required; title of the section heading that begins this chunk |
| `section_path` | list of strings | Required; breadcrumb of enclosing section titles from h1 down to the current heading |
| `anchor_id` | string or null | Anchor ID (e.g., `lab26`) from the `<a id="...">` preceding the section heading; null if no anchor present |

### 4.2 Chunk

| Field | Type | Constraints |
|-------|------|-------------|
| `text` | string | Required; plain-text content of the chunk (prose + code inline) |
| `code_blocks` | list of strings | Required; may be empty; Coq code blocks extracted from `<div class="code">` elements, preserving whitespace structure |
| `metadata` | ChunkMetadata | Required |
| `token_count` | positive integer | Required; approximate whitespace-split token count |

### 4.3 EducationSearchResult

| Field | Type | Constraints |
|-------|------|-------------|
| `text` | string | Required; chunk text |
| `code_blocks` | list of strings | Required |
| `metadata` | ChunkMetadata | Required |
| `score` | float | Required; cosine similarity score in [0, 1] |
| `location` | string | Required; human-readable breadcrumb (e.g., "LF > Basics > Proof by Simplification") |
| `browser_path` | string | Required; local file path with anchor (e.g., `~/software-foundations/lf/Basics.html#lab26`) |

## 5. Behavioral Requirements

### 5.1 HTMLChunker

#### chunk_file(html_path, volume)

- REQUIRES: `html_path` points to an existing HTML file generated by coqdoc. `volume` is one of the seven volume abbreviations.
- ENSURES: Returns a list of `Chunk` objects representing the sections of the page, in document order.
- Strips `<div id="header">`, `<div id="footer">`, `<div class="togglescript">`, and `<script>` elements before processing.
- Extracts volume title from `<div class="booktitleinheader">`.
- Extracts chapter name from `<h1 class="libtitle">`.
- Splits content on `<h1-h4 class="section">` boundaries.
- Each section accumulates `<div class="doc">` (prose) and `<div class="code">` (Coq) blocks.
- Prose is converted to plain text: HTML tags stripped, `&nbsp;` decoded, `<br/>` converted to newlines, inline code content preserved.
- Code blocks are extracted as separate strings with whitespace structure preserved.

> **Given** the file `software-foundations/lf/Basics.html`
> **When** `chunk_file(path, "lf")` is called
> **Then** returns a list of Chunk objects where each chunk has `metadata.volume == "lf"`, `metadata.chapter_file == "Basics.html"`, and `metadata.volume_title == "Logical Foundations"`

> **Given** an HTML file with 3 `<h1 class="section">` headings
> **When** `chunk_file` is called
> **Then** returns at least 3 chunks (more if sections are split for size)

#### Chunk size control

- If a chunk exceeds ~1000 tokens, split at `<div class="doc">` boundaries.
- If a chunk is under 100 tokens, merge with the following chunk (inheriting the next chunk's section title is acceptable; the merged chunk retains the earlier anchor_id).
- Exercise sections (h4 headings containing "Exercise") are kept as separate chunks even if under 100 tokens.

> **Given** a section with 1500 tokens
> **When** chunked
> **Then** produces two chunks, each under ~1000 tokens, split at a paragraph boundary

> **Given** a section with 50 tokens followed by a section with 200 tokens
> **When** chunked
> **Then** the two sections are merged into a single chunk

> **Given** an exercise section with 80 tokens
> **When** chunked
> **Then** the exercise remains a separate chunk (not merged)

#### chunk_corpus(sf_dir)

- REQUIRES: `sf_dir` is a directory containing volume subdirectories (`lf/`, `plf/`, etc.), each with `.html` files.
- ENSURES: Returns all chunks across all volumes. Skips non-content pages: `index.html`, `toc.html`, `coqindex.html`, `deps.html`.
- Volume abbreviation-to-directory mapping: `lf` = "Logical Foundations", `plf` = "Programming Language Foundations", `vfa` = "Verified Functional Algorithms", `qc` = "QuickChick", `secf` = "Security Foundations", `slf` = "Separation Logic Foundations", `vc` = "Verifiable C".

### 5.2 EducationEncoder

#### load(model_path, tokenizer_path)

- REQUIRES: `model_path` points to an existing INT8 ONNX model file. `tokenizer_path` points to an existing `tokenizer.json` file.
- ENSURES: Returns an `EducationEncoder` instance ready for inference. Loads the ONNX model via `onnxruntime.InferenceSession` with `CPUExecutionProvider`. Loads the tokenizer via `tokenizers.Tokenizer.from_file()`.
- On file not found: raises `ModelNotFoundError`.
- On invalid ONNX format: raises `ModelLoadError`.

#### encode(text)

- REQUIRES: `text` is a non-empty string. Encoder is loaded.
- ENSURES: Returns an L2-normalized float32 numpy array of dimension 384. The same input always produces the same output (deterministic).
- On text longer than 256 tokens: input is truncated to 256 tokens before encoding. No error is raised.
- Processing: tokenize → ONNX inference → mean pooling over token embeddings (masked by attention) → L2 normalize.

> **Given** encoder is loaded
> **When** `encode("proof by induction on natural numbers")` is called
> **Then** returns a 384-dim float32 vector with L2 norm ≈ 1.0

#### encode_batch(texts)

- REQUIRES: `texts` is a non-empty list of non-empty strings. Encoder is loaded.
- ENSURES: Returns a list of 384-dim L2-normalized float32 vectors. Order preserved. Each vector is identical to calling `encode` individually.

#### model_hash()

- ENSURES: Returns the SHA256 hex digest of the ONNX model file. Used to verify consistency between the model and pre-computed embeddings.

### 5.3 EducationStorage

#### create(db_path)

- REQUIRES: `db_path` is a writable file path.
- ENSURES: Creates a new SQLite database with the schema defined in the architecture document (tables: `chunks`, `education_embeddings`, `education_meta`, `chunks_fts`). If the file already exists, it is deleted and recreated.

#### write_chunks(db_path, chunks)

- REQUIRES: `db_path` points to an existing education database. `chunks` is a non-empty list of `Chunk` objects.
- ENSURES: Inserts all chunks into the `chunks` table. Returns a list of assigned chunk IDs in the same order. Populates the `chunks_fts` FTS5 index.

#### write_embeddings(db_path, chunk_ids, vectors)

- REQUIRES: `db_path` points to an existing education database. `chunk_ids` and `vectors` have the same length. Each vector is a 384-dim float32 array.
- ENSURES: Inserts all embeddings into `education_embeddings`. Vectors are stored as raw float32 bytes (1536 bytes per vector).

#### write_meta(db_path, metadata)

- REQUIRES: `db_path` points to an existing education database. `metadata` is a dict of string key-value pairs.
- ENSURES: Inserts or replaces all key-value pairs in `education_meta`.

#### load_chunks(db_path)

- REQUIRES: `db_path` points to an existing education database.
- ENSURES: Returns a dict mapping chunk ID to `Chunk` object for all chunks in the database.

#### load_embeddings(db_path)

- REQUIRES: `db_path` points to an existing education database with populated `education_embeddings` table.
- ENSURES: Returns a tuple `(matrix, id_map)` where `matrix` is a float32 numpy array of shape `(N, 384)` and `id_map` is an int array of shape `(N,)` mapping row indices to chunk IDs.

### 5.4 EducationRAG

#### __init__(db_path, model_path, tokenizer_path)

- REQUIRES: All paths point to existing files.
- ENSURES: Loads the encoder, loads embeddings into an in-memory `EmbeddingIndex`, loads chunk metadata. The instance is ready for queries.
- On missing or invalid files: logs a warning and sets `is_available()` to False.

#### search(query, limit=5, volume_filter=None)

- REQUIRES: `query` is a non-empty string. `limit` is a positive integer.
- ENSURES: Encodes the query, performs cosine similarity search against the embedding index, and returns up to `limit` `EducationSearchResult` objects ranked by descending score.
- When `volume_filter` is provided: filters results to chunks matching the specified volume.
- The `location` field is formatted as `"{VOLUME_ABBREV} > {chapter} > {section_title}"` with the volume abbreviation uppercased.
- The `browser_path` field is formatted as `"~/software-foundations/{volume}/{chapter_file}#{anchor_id}"`. When `anchor_id` is null, the fragment is omitted.

> **Given** the education database is loaded
> **When** `search("induction on natural numbers", limit=3)` is called
> **Then** returns up to 3 results, each with a score, location string, browser path, and chunk text

> **Given** `volume_filter="lf"`
> **When** `search("rewriting", volume_filter="lf")` is called
> **Then** all returned results have `metadata.volume == "lf"`

#### is_available()

- ENSURES: Returns True if the encoder, embedding index, and chunk metadata are all loaded. Returns False otherwise.

### 5.5 MCP Tool Handler

#### handle_education_context(ctx, query, limit=3, volume=None)

- REQUIRES: `query` is a string. `limit` defaults to 3, clamped to [1, 10]. `volume`, when provided, must be one of the seven volume abbreviations.
- ENSURES when `ctx.education_rag` is available: Calls `ctx.education_rag.search(query, limit, volume)` and returns a formatted success response with results as JSON text.
- ENSURES when `ctx.education_rag` is None or unavailable: Returns an error response with code `EDUCATION_UNAVAILABLE`.

> **Given** education RAG is loaded
> **When** `handle_education_context(ctx, query="induction", limit=3)` is called
> **Then** returns a success response with up to 3 results, each containing text, code_blocks, location, browser_path, and score

> **Given** education RAG is not loaded
> **When** `handle_education_context(ctx, query="induction")` is called
> **Then** returns an error response with code EDUCATION_UNAVAILABLE

### 5.6 Build Pipeline

#### build_education_db(sf_dir, output_path, model_path, tokenizer_path)

- REQUIRES: `sf_dir` is a directory containing SF volume subdirectories. `output_path` is a writable file path. Model and tokenizer paths point to existing files.
- ENSURES: Creates `output_path` as a populated education database containing all chunks and embeddings from the SF corpus.
- Steps: (1) chunk the corpus, (2) create database, (3) write chunks, (4) load encoder, (5) encode chunks in batches of 64, (6) write embeddings, (7) write metadata (schema_version, model_hash, sf_version, build_date, chunk_count).
- If `output_path` already exists, it is deleted and recreated.

> **Given** a valid SF directory with 7 volumes
> **When** `build_education_db(sf_dir, output, model, tokenizer)` is called
> **Then** creates a SQLite database at `output` with chunks, embeddings, FTS index, and metadata tables populated
